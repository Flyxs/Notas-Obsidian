# Métodos para resolução de colisões

## 1. Encadeamento Separado

``` mermaid
graph LR
	0 --> Null
	1 --> Null
	...
	n-1 --> Null
	n --> Null
```

Cada posição é uma lista encadeada com as colisões daquela posição.

### TAD

``` c
typedef struct celula{
	int chave;
	struct celula *prox;
}celula;

typedef struct hash{
	celula **tab;
	int m;
}hash;

hash *cria_hash(int m);
void inserir(hash *h, int chave);
void remover(hash *h, int chave);
celula *buscar(hash *h, int chave);
void destruir_hash(hash *h);
int fhash(hash *h, int chave);
```
``` c
int fhash(hash *h, int chave){
	return chave % h->m;
}

hash *cria_hash(int m){
	hash *m = malloc(sizeof(hash));
	h->tab = malloc(m * sizeof(celula *));
	for(int i = 0; i < m; i++){
		h->tab[i] = malloc(sizeof(celula));
		h->tab[i] -> prox = NULL;
	}
	h->m = m;
	return h;
}

void inserir(hash *h, int chave){
	int pos = fhash(h, chave);
	inserir_lista(h->tab[pos], chave);
	# |-> 1. Verifica de a chave está presente na lista.
	# |-> 2. Inserir a chave na lista, caso não esteja presente.
	# Sugestão - manter a lista ordenada
	#          - substitua lista por ABBRNE
}

void remover(hash *h, int chave){
	int pos = fhash(h, chave);
	remove_lista(h->tab[pos], chave);
	# |-> Remove o nó com chave da lista h->tab[pos]
}

celula *buscar(hash *h, int chave){
	int pos = fhash(h, chave);
	return buscar_lista(h->tab[pos], chave);
	# |-> verifica se chave está presente na lista h->tab[pos]
	# * Se sim, retorna o endereço do nó
	# * se não, retorna NULL
}

void destruir_hash(hash *h){
	for(int i = 0; i < m; i++) {
		free(h->tab);
		free(h);
	}
}
```

## 2. Endereçamento aberto

Implementação com *vetores*:
* A função de hash calcula a posição.
* Se houver colisão, insere numa *posição livre* do vetor.

Como decidir em qual posição inserir no caso de colisões?
### 1. Sondagem Linear 

Insere na próxima posição "livre" à direita, módulo M.

**Exemplo**: hash com tamanho 7, função: método da divisão

``` mermaid
graph TD
	primeiro["[ -- , -- , -- , -- , -- , -- , -- ]"] -->|insere 40| segundo["[ -- , -- , -- , -- , -- , 40 , -- ]"]
	segundo -->|insere 68| terceiro["[-- , -- , -- , -- , -- , 40 , 68]"]
	terceiro -->|insere 70| quarto["[70 , -- , -- , -- , -- , 40 , 68]"]
	quarto -->|insere 50| quinto["[70 , 50 , -- , -- , -- , 40 , 68]"]
	quinto -->|insere 33| sexto["[70 , 50 , 33 , -- , -- , 40 , 68]"]
	sexto -->|remove 50| setimo["[70 , 33 , -- , -- , -- , 40 , 68]"]
```

Na sondagem linear:

1. Inserção de um objeto com chave `ch`:
	* Tenta inserir o objeto na posição fhash(ch)
	* Em um caso de colisão, procura procura pela primeira posição "livre" a direita, modulo M.

2. Busca por um objeto com chave `ch`:
	* Começa na posição `fhash(ch)`
	* Procura por todas à direita, modulo M, até encontrar uma posição "livre".

3. Remoção de um objeto com chave `ch`:
	* Busca pelo objeto
	* Marca a posição do objeto como "livre"
	* Reinsere na tabela todos os elementos à direita, até encontrar uma posição "livre".
		* para cada objeto que, ao ser reinserido, mudar de posição, não esquecer de marcar a posição antiga como "livre".

>[!tip] Observação:
> (a) A posição "livre" deve ser um valor numérico que não pertença ao domínio das chaves. *Ex.:* Se um dos objetos fossem aluno, livre poderia ser o zero (já que ninguém tem matricula zero).
> (b) Se houver necessidade de redimensionar a tabela:
> 	Escolher novo primo maior que 2 * m.
> 	Reinserir todos os elementos na tabela depois de redimensionar.

### 2. Hashing Duplo

É igual à sondagem linear, só que ao invés de procurar pela proxima posição livre saltando uma posição à direita, salta fhash2, sendo esta uma segunda função hash.

>[!warning] Importante
>fhash2 precisa ser coprimo com M
>*Ex.:* M potencia de 2 fhash2 sempre impar
>     M primo e fhash2 < m
